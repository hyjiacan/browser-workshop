================================================================================
                               核心概念
================================================================================

【概述】
  本文档介绍 bws 的核心概念，帮助用户理解工具的工作原理和设计思想。

【概念列表】

  ┌─────────────────┬────────────────────────────────────────────────────┐
  │ 概念             │ 简要说明                                           │
  ├─────────────────┼────────────────────────────────────────────────────┤
  │ 版本             │ 浏览器的具体版本号，如 120.0.6099.109              │
  │ 浏览器类型       │ chrome / firefox / chromium                        │
  │ 渠道             │ 发布渠道：stable, beta, dev, canary, esr           │
  │ 数据源           │ 提供版本信息和安装包的来源                         │
  │ 仓库             │ 本地存储已安装浏览器版本的目录                     │
  │ 离线源           │ 通过 bws serve 提供的本地下载源                    │
  │ Profile          │ 浏览器的数据目录，包含历史、Cookie、密码等         │
  │ 系统浏览器       │ 操作系统已安装的浏览器（非 bws 管理）              │
  │ 版本别名         │ latest, stable, beta, dev, canary, esr, system     │
  │ 浏览器短别名     │ gc=chrome, ff=firefox, cm=chromium                 │
  └─────────────────┴────────────────────────────────────────────────────┘

【概念详解】

  --- 版本（Version） ---
  浏览器的完整版本号，格式通常为：主版本号.次版本号.构建号.补丁号
  例如：120.0.6099.109

  支持部分版本号匹配：
  * chrome@120     - 匹配 120.x 的最新版本
  * chrome@120.0   - 匹配 120.0.x 的最新版本

  --- 浏览器类型 ---
  bws 支持三种浏览器：
  * chrome     - Google Chrome（闭源）
  * chromium   - Chromium 开源项目
  * firefox    - Mozilla Firefox

  浏览器短别名可在所有命令中使用：
  * gc = chrome
  * ff = firefox
  * cm = chromium

  --- 渠道（Channel） ---
  浏览器的发布渠道，决定了版本的稳定性和功能更新频率：
  * stable（稳定版）：经过充分测试，适合日常使用
  * beta（测试版）：包含下一版本的新功能，稳定性较好
  * dev（开发版）：包含最新功能，可能存在不稳定因素
  * canary（金丝雀版）：每日构建，最新但最不稳定
  * esr（长期支持版）：Firefox 特有的延长支持版本

  --- 数据源（Source） ---
  提供版本列表和安装包的来源：
  * Serve 离线源：通过 bws serve 搭建的局域网 HTTP 源
  * Omaha 协议：Chrome/Chromium 官方更新协议
  * Firefox API：Mozilla Product Details API

  参见 "bws help sources" 获取详细信息。

  --- 仓库（Repository） ---
  本地存储已安装浏览器版本的目录结构。每个版本独立存放，便于管理和切换。

  数据目录结构示例：
  ┌──────────────────────────────────────────────────┐
  │ bws-data/                                        │
  │ ├── versions/                                    │
  │ │   ├── chrome/                                  │
  │ │   │   ├── 120.0.6099.109/                      │
  │ │   │   └── 121.0.6167.85/                       │
  │ │   └── firefox/                                 │
  │ │       └── 121.0/                               │
  │ ├── profiles/                                    │
  │ ├── plugins/                                     │
  │ └── config.json                                  │
  └──────────────────────────────────────────────────┘

  --- Profile ---
  浏览器的数据目录，包含历史记录、Cookie、密码、扩展等。
  bws 为每个版本和每个命名 Profile 创建独立的数据目录，实现完全隔离。

  --- 版本别名 ---
  用于简化版本指定的特殊名称：
  * latest   - 最新稳定版
  * stable   - 稳定版（同 latest）
  * beta     - 测试版
  * dev      - 开发版
  * canary   - 金丝雀版
  * esr      - 长期支持版（Firefox）
  * system   - 系统已安装的版本

【工作流程】

  典型使用流程：
  1. 查询已安装版本：bws ls
  2. 查看远程版本：  bws ls -R chrome
  3. 安装版本：      bws i chrome@120
  4. 运行版本：      bws r chrome@120
  5. 设置默认：      bws u chrome@120
  6. 管理版本：      bws ls / bws rm chrome@120
