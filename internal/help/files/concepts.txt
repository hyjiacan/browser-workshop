================================================================================
                               核心概念
================================================================================

【概述】
  本文档介绍 BWS 的核心概念，帮助用户理解工具的工作原理和设计思想。

【概念列表】

  ┌─────────────────┬────────────────────────────────────────────────────┐
  │ 概念             │ 简要说明                                           │
  ├─────────────────┼────────────────────────────────────────────────────┤
  │ 版本             │ 浏览器的具体版本号，遵循语义化版本规则              │
  │ 渠道             │ 浏览器的发布渠道，如稳定版、测试版、开发版          │

  │ 缓存             │ 下载的安装包临时存储，用于加速重复安装              │
  │ 配置             │ 工具的全局设置，控制各项行为                       │
  └─────────────────┴────────────────────────────────────────────────────┘

【概念详解】

  --- 版本（Version） ---
  浏览器的完整版本号，格式为：主版本号.次版本号.构建号.补丁号
  例如：129.0.6668.100

  版本号各段含义：
  * 主版本号：重大功能变更，通常包含新 UI 或架构调整
  * 次版本号：新功能添加，保持向后兼容
  * 构建号：内部构建编号，用于追踪构建来源
  * 补丁号：Bug 修复和安全更新

  --- 渠道（Channel） ---
  浏览器的发布渠道，决定了版本的稳定性和功能更新频率：
  * stable（稳定版）：经过充分测试，适合日常使用
  * beta（测试版）：包含下一版本的新功能，稳定性较好
  * dev（开发版）：包含最新功能，可能存在不稳定因素

  --- 数据源（Source） ---
  提供版本列表和安装包的来源。参见 "bws help sources" 获取详细信息。

  --- 仓库（Repository） ---
  本地存储已安装浏览器版本的目录结构。每个版本独立存放，
  便于管理和切换。

  仓库结构示例：
  ┌──────────────────────────────────────────────────┐
  │ 仓库根目录/                                      │
  │ ├── 129.0.6668.100/                             │
  │ │   ├── chrome.exe                               │
  │ │   └── ...                                      │
  │ ├── 128.0.6613.120/                             │
  │ │   ├── chrome.exe                               │
  │ │   └── ...                                      │
  │ └── default -> 129.0.6668.100  （符号链接）       │
  └──────────────────────────────────────────────────┘

  --- 离线源（Offline Source） ---
  通过 bws serve 命令启动的 HTTP 服务，将本地仓库中的版本
  以 HTTP 方式共享给局域网内的其他设备。

  --- 缓存（Cache） ---
  下载的浏览器安装包会缓存在本地，当再次安装相同版本时
  直接从缓存读取，避免重复下载。缓存可通过 bws repo clean
  命令清理。

  --- 配置（Configuration） ---
  工具的全局设置，存储在配置文件中。参见 "bws help config"
  获取详细的配置项说明。

【工作流程】

  典型使用流程：
  1. 查询版本：bws ls
  2. 安装版本：bws install <版本号>
  3. 切换版本：bws repo default <版本号>
  4. （可选）搭建离线源：bws serve
  5. 管理版本：bws repo list / remove
