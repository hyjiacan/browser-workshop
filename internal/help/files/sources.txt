================================================================================
                              数据源说明
================================================================================

【概述】
  bws 支持从多个数据源获取浏览器版本信息及安装包。数据源分为
  离线源（serve）和在线源（Omaha / Firefox Product Details）两类。

【数据源类型】

  ┌───────────────────┬──────────┬──────────────────────────────────────────┐
  │ 数据源             │ 类型     │ 说明                                     │
  ├───────────────────┼──────────┼──────────────────────────────────────────┤
  │ Serve 离线源       │ 离线源   │ 通过 bws serve 搭建的本地/局域网 HTTP 源 │
  │ Omaha 协议         │ 在线源   │ Chrome/Chromium 官方更新协议             │
  │ Firefox 在线源     │ 在线源   │ Mozilla Product Details API              │
  └───────────────────┴──────────┴──────────────────────────────────────────┘

【数据源详情】

  --- Serve 离线源 ---
  通过 bws serve 命令搭建的本地或局域网 HTTP 下载源。

  配置方法：
    bws cfg set source http://<服务器地址>:<端口>
    bws cfg set source-serve true

  适用场景：
  * 企业内部分发，避免重复下载
  * 离线环境下的版本管理
  * 需要特定版本定制的场景

  --- Omaha 协议（Chrome/Chromium）---
  Google Chrome 的官方更新协议，用于获取版本信息和下载链接。

  配置方法：
    bws cfg set source-omaha true

  支持的浏览器：
  * chrome
  * chromium

  --- Firefox 在线源 ---
  Mozilla Product Details API，获取 Firefox 版本信息。

  配置方法：
    bws cfg set source-firefox-ftp true

  支持的浏览器：
  * firefox

【数据源开关】

  通过 config 命令控制各数据源的启用/禁用：

    bws cfg set source-serve true     # 启用 Serve 离线源
    bws cfg set source-omaha true     # 启用 Omaha 在线源
    bws cfg set source-firefox-ftp true # 启用 Firefox 在线源

【数据源优先级】

  当同时配置多个数据源时，优先级如下：
  1. 离线源（Serve 源，如果已配置且启用）
  2. Omaha 在线源（Chrome/Chromium）
  3. Firefox 在线源

  查询特定浏览器时，仅从支持该浏览器的数据源获取信息。

【注意事项】
  * 首次使用远程源时可能需要下载版本索引信息，耗时取决于网络状况。
  * 建议在内网环境使用 Serve 离线源，减少对外网依赖。
  * 所有在线源都支持通过 proxy 配置使用代理服务器。
