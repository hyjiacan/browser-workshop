================================================================================
                              数据源说明
================================================================================

【概述】
  BWS 支持从多个数据源获取浏览器版本信息及安装包。数据源分为
  官方源和镜像源两类，用户可以根据网络环境选择合适的数据源。

【数据源类型】

  ┌─────────────┬──────────┬──────────────────────────────────────────────┐
  │ 数据源       │ 类型     │ 说明                                         │
  ├─────────────┼──────────┼──────────────────────────────────────────────┤
  │ google      │ 官方源   │ Google Chrome 官方发布渠道                   │
  │ nih         │ 镜像源   │ 国内镜像源，提供更快的下载速度               │
  │ auto        │ 自动选择 │ 根据网络环境自动选择最优数据源               │
  │ custom      │ 自定义源 │ 用户通过 serve 命令搭建的离线下载源          │
  └─────────────┴──────────┴──────────────────────────────────────────────┘

【数据源详情】

  --- google（官方源） ---
  Google Chrome 的官方发布渠道，直接从 Google 服务器获取版本信息
  和安装包。适合具有良好国际网络访问环境的用户。

  优势：
  * 版本更新最及时
  * 数据最权威准确
  * 支持所有发布渠道（stable、beta、dev）

  劣势：
  * 国内访问速度可能较慢
  * 可能需要代理访问

  --- nih（镜像源） ---
  国内镜像源，为国内用户提供加速下载服务。定期从官方源同步数据。

  优势：
  * 国内访问速度快
  * 无需代理
  * 下载稳定

  劣势：
  * 数据同步可能存在延迟
  * 可能不支持所有历史版本

  --- auto（自动选择） ---
  自动模式，程序会根据当前网络环境自动选择最优的数据源。

  选择逻辑：
  1. 首先检测与官方源的连通性
  2. 如果官方源不可达或速度慢，切换到镜像源
  3. 如果配置了自定义源地址，优先使用自定义源

  --- custom（自定义源） ---
  通过 bws serve 命令搭建的本地或局域网 HTTP 下载源。

  配置方法：
  bws config set source-url http://<服务器地址>:<端口>

  适用场景：
  * 企业内部分发，避免重复下载
  * 离线环境下的版本管理
  * 需要特定版本定制的场景

【数据源切换】
  临时切换（单次命令有效）：
  bws install <版本号> --source google

  永久切换（修改配置）：
  bws config set default-source nih

【数据源优先级】
  当同时配置多个数据源时，优先级如下：
  1. 命令行指定的数据源（最高优先级）
  2. 自定义源（source-url）
  3. 默认数据源（default-source 配置项）
  4. 自动选择模式（最低优先级）
